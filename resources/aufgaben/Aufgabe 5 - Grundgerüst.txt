Sie haben die Anforderungen an Ihr Semesterprojekt ermittelt (und im Laufe der letzen Wochen angepasst, so dass diese die Ihnen aus dem letzten Semester bekannten Qualitätskriterien erfüllen). Sie haben eine erste Archtektur entworfen. Vielleicht haben Sie diese sogar inzwischen ebenfalls verbessert.

Damit haben Sie (fast) alles, was Sie für eine Implementierung benötigen.

Ihre Aufgabe besteht nun darin, auch zu zeigen, dass Sie auf der eigentlichen Entwicklungsebene bereit sind. Sonst war alles "nur Geschwätz", ohne Wertschöpfung.

Erstellen Sie eine allererste Version der von Ihnen geplanten Software. Diese muss nicht "vollständig" sein. Entwickeln Sie eine App für ein mobiles Endgerät, dann reicht eine App, die ein "Ahoi!" auf dem mobilen Endgerät darstellt. Entwickeln Sie einen Web-Service, dann reicht ein erster Web-Server, der die geplante Startseite als HTML-Antwort liefert; etwas CSS könnte ausgeliefert werden, um für etwas Ästhetik zu sorgen. Sofern für Ihre selbst definierte Problemstellung relevan, sollten Sie auch eine Liste von Zetteln aus dem Zettelstore auf der ersten Webseite darstellen (es reicht Zettel-ID und -titel). Wenn die Liste von Zetteln nicht zu Ihrer Problemstellung passt, dann muss die Softwareversion mindestens igendwie auf den Zettelstore mittels seiner API (inhaltlich sinnvoll) zugreifen und im Sinne des Projekts so ermittelte Daten ausgeben oder dort abliefern.

Sie haben schon im letzten Semester gelernt, wie man eine Web-Anwendung erstellt. Neu ist maximal, wie Sie auf den Zettelstore technisch zugreifen können. Die theoretischen Grundlagen wurden in den letzten Vorlesungsterminen gelegt. Die Dokumentation zur API des Zettelstore finden Sie unter https://zettelstore.de/manual/h/00001012000000. Wie man einen Web-Request von Python aus sendet (und die Antwort empfängt) wurde in der Theorie kurz angesprochen. Hilfreich sind hierbei das von Python mitgelieferte Paket "urllib.request" (https://docs.python.org/3/library/urllib.request.html) oder das zusätzlich zu installierende Paket "requests" (via "pip install requests", https://pypi.org/project/requests/). Sollten Sie Go als Programmiersprache verwenden, können Sie die Go-Standardbibliothek verwenden (Paket net/http, https://pkg.go.dev/net/http) oder Sie verwenden den Zettelstore-Client. Im letzteren Fall ist die Verarbeitung der Daten besonders einfach, allerdings müssen Sie neben Git auch "Fossil" installieren (https://fossil-scm.org), um dann den Zettelstore-Client per "GOVCS=t73f.de:fossil go get -v -x t73f.de/r/zsc@be388711ad2a99ef" Ihrem Projekt bekannt zu geben.
